# 文件搜索与常用启动项能力扩展需求文档

## 1. 文档信息

- 文档名称：文件搜索与常用启动项能力扩展需求文档
- 文档类型：产品需求文档（PRD）
- 适用范围：`PackageManager` 内置 `文件搜索` 与 `常用启动项`
- 文档目的：为后续技术方案、排期拆分、界面迭代提供统一需求基线

## 2. 背景

当前项目中的 `文件搜索` 功能已经具备基础搜索能力，但仍处于雏形阶段，主要问题是：

- 搜索结果可查看，但“下一步动作”较少，使用链路偏断裂
- 常见高频操作仍需依赖右键或手工跳转，效率不够高
- 缺少与 `PackageManager` 自身场景深度结合的定制化能力
- 与 `Everything` 相比，虽然功能尚弱，但我们具备可持续定制和深度集成的优势

当前 `常用启动项` 功能已经具备搜索、添加、编辑、启动等基础能力，但仍有以下提升空间：

- 更偏“收藏夹”，还不够像一个高频启动面板
- 启动前后的辅助动作不足
- 缺少批量管理、分组、排序、快捷触达等效率能力

因此，需要将这两个功能从“可用”升级为“高频可依赖”，形成一套适合团队工作流的轻量桌面效率工具。

## 3. 产品目标

### 3.1 总体目标

- 让 `文件搜索` 成为团队日常定位文件、目录、脚本、日志、配置的高频入口
- 让 `常用启动项` 成为团队日常启动工具、脚本、项目入口的高频面板
- 两者形成互补关系：
  - `文件搜索` 负责“临时查找与快速处理”
  - `常用启动项` 负责“固定入口与稳定启动”

### 3.2 核心目标

- 降低用户完成“搜到文件后继续处理”的操作成本
- 降低用户重复打开常用工具、常用目录、常用脚本的操作成本
- 通过快捷键、右键动作、批量动作、上下文动作，提升单次操作效率
- 逐步形成区别于 `Everything` 的团队定制能力

## 4. 设计原则

- 不追求一次性对标 `Everything` 全量能力
- 优先做高频、闭环、可积累的数据型能力
- 优先做符合团队真实使用场景的动作
- 同一功能尽量支持“鼠标路径”和“键盘路径”
- 所有新增能力都应尽量减少思考成本，避免把简单操作做复杂

## 5. 目标用户与典型场景

### 5.1 目标用户

- 日常需要频繁查找安装目录、脚本、配置文件的开发人员
- 需要快速打开工具、项目、日志目录的实施或运维人员
- 需要频繁在多个工作目录间切换的高级用户

### 5.2 典型场景

- 搜索某个 exe、bat、ps1 后直接启动
- 搜索某个目录后直接打开、复制路径、加入常用入口
- 搜索日志文件后直接打开所在目录或复制完整路径
- 搜索配置文件后直接用指定编辑器打开
- 找到经常使用的工具后，一键加入 `常用启动项`
- 从 `常用启动项` 中快速启动工具、脚本、项目目录

## 6. 文件搜索能力扩展需求

### 6.1 核心定位

`文件搜索` 不仅要“找到”，还要“立刻处理”。

### 6.2 快捷操作扩展

以下为优先建议补齐的常用快捷操作。

#### 6.2.1 基础键盘操作

- `Enter`：打开选中项
- `Ctrl+Enter`：打开所在目录
- `Shift+Enter`：以管理员身份运行或打开
- `Ctrl+C`：复制完整路径
- `Ctrl+Shift+C`：仅复制文件名
- `Alt+Enter`：打开属性
- `Delete`：进入删除确认流程
- `F2`：重命名
- `F5`：刷新当前查询结果
- `Ctrl+L`：聚焦到搜索框并选中全部关键词
- `Esc`：隐藏窗口

#### 6.2.2 搜索结果右键菜单

- 打开
- 打开所在目录
- 复制完整路径
- 复制文件名
- 复制父目录路径
- 以管理员身份运行
- 用指定程序打开
- 打开终端到当前目录
- 加入常用启动项
- 加入常用目录或收藏入口
- 查看属性

#### 6.2.3 批量操作

- 批量复制路径
- 批量打开所在目录
- 批量加入常用启动项
- 批量导出搜索结果
- 批量删除或移动到回收站

### 6.3 搜索体验优化

#### 6.3.1 查询能力

- 支持最近搜索历史
- 支持固定搜索条件模板
- 支持常见搜索语法提示
- 支持按类型快速过滤：`exe`、`脚本`、`文件夹`、`日志`、`配置`
- 支持按范围过滤：桌面、下载、文档、指定目录、自定义工作区
- 支持“仅显示可启动文件”
- 支持“仅显示目录”
- 支持“仅显示最近修改”

#### 6.3.2 排序与视图

- 支持按名称、路径、修改时间、大小排序
- 支持置顶最近使用结果
- 支持记忆上次排序方式
- 支持紧凑模式与详细模式

#### 6.3.3 上下文增强

- 在结果中高亮命中片段
- 明确标记文件类型和可执行性
- 目录结果与文件结果视觉区分
- 对常用目录、常用工具、最近打开项做视觉标记

### 6.4 团队定制化能力

这是后续区别于 `Everything` 的重点。

- 支持针对团队工作目录的预置搜索范围
- 支持内置“常用关键词收藏”
- 支持识别特定文件类型并给出定制动作
  - 如：`rvt` 文件支持直接进入清理相关流程
  - 如：日志文件支持快速打开日志查看器
  - 如：解决方案文件支持直接打开 IDE
- 支持对搜索结果打标签
- 支持将某类结果快速加入 `常用启动项`

### 6.5 数据积累能力

- 记录最近打开文件
- 记录最近复制路径
- 记录最近搜索词
- 记录高频搜索结果点击次数
- 基于历史行为做排序优化

## 7. 常用启动项能力扩展需求

### 7.1 核心定位

`常用启动项` 应从“程序收藏列表”升级为“工作台启动面板”。

### 7.2 快捷操作扩展

#### 7.2.1 基础键盘操作

- `Enter`：启动选中项
- `Ctrl+Enter`：打开所在目录
- `F2`：编辑启动项
- `Delete`：删除启动项
- `Ctrl+Up` / `Ctrl+Down`：调整排序
- `Ctrl+D`：复制一份启动项配置
- `Ctrl+C`：复制启动项路径
- `Esc`：隐藏窗口

#### 7.2.2 列表操作

- 置顶常用项
- 自定义排序
- 分组显示
- 折叠分组
- 搜索过滤
- 最近启动记录
- 启动次数统计

#### 7.2.3 启动增强

- 普通启动
- 管理员启动
- 启动前确认
- 启动后自动隐藏窗口
- 启动失败时展示明确错误信息
- 支持附带参数模板
- 支持多项连续启动

### 7.3 管理能力

- 支持拖拽添加
- 支持从 `文件搜索` 直接添加
- 支持批量导入启动项
- 支持批量导出启动项
- 支持启动项备注
- 支持图标展示
- 支持检测路径失效并提示修复

### 7.4 分组与工作流能力

- 支持按场景分组
  - 开发工具
  - 项目入口
  - 运维脚本
  - 临时工具
- 支持“一组全部启动”
- 支持按用户习惯收起不常用分组
- 支持收藏星标

## 8. 文件搜索与常用启动项的联动需求

这是两个功能最值得做深的方向。

### 8.1 从搜索到收藏

- 文件搜索结果支持“一键加入常用启动项”
- 加入时自动识别名称、路径、类型
- 对脚本、快捷方式、可执行文件采用不同默认策略

### 8.2 从收藏到搜索

- 常用启动项支持“在文件搜索中定位原文件”
- 失效项支持“重新搜索并重新绑定”

### 8.3 双向数据能力

- 根据文件搜索点击频率，推荐加入常用启动项
- 根据常用启动项启动失败，推荐重新搜索替换路径
- 根据用户最近搜索和启动记录推荐候选项

## 9. 分阶段实施建议

### 9.1 第一阶段：高频闭环

优先补齐最常用、最有感知的效率动作。

- 文件搜索结果支持打开、打开目录、复制路径、加入常用启动项
- 文件搜索支持基础快捷键
- 常用启动项支持启动、编辑、删除、排序
- 两个窗口都支持统一键盘行为
- 常用启动项支持启动后自动隐藏可选项

### 9.2 第二阶段：管理增强

- 文件搜索支持历史、过滤器、排序记忆
- 常用启动项支持分组、拖拽排序、批量管理
- 两者联动支持“从失效项重新搜索修复”

### 9.3 第三阶段：团队定制

- 针对工作目录、日志、项目文件、脚本做场景化动作
- 引入行为统计和推荐
- 引入搜索模板与工作流入口

## 10. 非目标

以下内容暂不作为当前需求文档的首要目标。

- 完整替代 `Everything`
- 一次性支持所有搜索语法和索引能力
- 引入过重的外部依赖导致主程序体积明显膨胀
- 把文件搜索做成复杂资源管理器

## 11. 验收标准建议

### 11.1 文件搜索

- 用户从搜索到打开目标文件的平均操作步数明显下降
- 用户从搜索结果加入常用启动项的路径足够直接
- 常见高频动作可通过键盘快速完成
- 搜索结果右键菜单覆盖主要后续动作

### 11.2 常用启动项

- 用户能更快地启动固定工具和脚本
- 启动项管理成本低，支持编辑、排序、分组
- 失效项可被识别并具备修复路径

### 11.3 联动

- 用户能从文件搜索平滑进入常用启动项
- 用户能从常用启动项反向定位原始文件
