# DevKit 代码工作区优化及功能扩展建议方案

## 一、代码工作区核心增强（高优先级）

### 1.1 Git/SVN 状态实时显示 ⭐⭐⭐⭐⭐
**现状问题**: 当前代码工作区只是一个"启动器"，看不到任何仓库状态信息（分支、未提交变更、远程同步状态等）

**建议方案**:
- 在仓库列表中增加列：
  - **当前分支** (Git: `git branch --show-current`, SVN: `svn info --show-item relative-url`)
  - **状态指示器** (绿色=干净, 黄色=有未提交, 红色=冲突)
  - **未提交变更数** (Git: `git status --porcelain | wc -l`, SVN: `svn status | wc -l`)
  - **远程同步状态** (Git: ahead/behind commits, SVN: revision差异)
- 后台异步刷新（每30秒或手动刷新）
- 点击状态可快速查看变更文件列表

**增效价值**: 
- 无需打开IDE即可掌握所有仓库状态
- 快速识别哪些仓库有待提交工作
- 避免遗忘未推送的提交

---

### 1.2 快速操作面板 ⭐⭐⭐⭐⭐
**现状问题**: 常见Git操作（pull, push, stash, checkout）都需要打开IDE或终端

**建议方案**:
在每个仓库行增加快捷操作按钮（右键菜单或下拉菜单）：
- **Pull** - 拉取最新代码
- **Push** - 推送当前分支
- **Stash** - 暂存当前变更
- **Checkout Branch** - 切换分支（弹出分支列表）
- **Create Branch** - 新建分支
- **Sync All** - 批量拉取所有仓库

操作结果通过Toast通知反馈，失败时显示详细错误。

**增效价值**:
- 批量同步所有仓库（早上上班第一件事）
- 快速切换分支无需打开IDE
- 减少80%的终端操作

---

### 1.3 搜索与智能过滤 ⭐⭐⭐⭐
**现状问题**: 仓库列表无搜索功能，仓库多时难以定位

**建议方案**:
- 顶部增加搜索框，支持：
  - 仓库名称模糊匹配
  - 路径匹配
  - 备注内容搜索
  - 拼音首字母搜索（复用QuickLaunch的PinyinMatcher）
- 智能过滤器：
  - "有未提交变更"
  - "最近7天使用过"
  - "特定分支"（如所有在feature分支的）
  - "Git仓库" / "SVN仓库"
- 保存常用过滤器为预设

**增效价值**:
- 50+仓库时快速定位
- 快速找到所有待提交的仓库

---

### 1.4 VS Code 集成 ⭐⭐⭐⭐
**现状问题**: 有Cursor按钮但没有VS Code按钮，而AppSettings已有VsCodePath字段

**建议方案**:
- 增加"VS Code"按钮，逻辑类似Cursor
- 支持打开特定文件夹或工作区文件（`.code-workspace`）
- 如果检测到`.code-workspace`文件，优先使用它打开

**增效价值**:
- VS Code是最常用的编辑器，必须支持
- 工作区文件可以保存多根目录配置

---

### 1.5 AI 提交增强 ⭐⭐⭐⭐
**现状问题**: 
- Claude/Codex提交都是硬编码的危险模式（`--dangerously-skip-permissions`）
- 没有提交历史记录
- 无法查看AI生成的提交信息

**建议方案**:
- **安全级别可配置**: 在设置中增加"AI提交安全级别"（安全/标准/快速），映射到不同的CLI参数
- **提交历史面板**: 记录每次AI提交的：
  - 时间戳
  - 仓库名
  - AI引擎（Claude/Codex）
  - 生成的提交信息
  - 是否成功推送
- **提交信息预览**: AI生成后先显示预览窗口，用户确认后再执行
- **模板管理**: 支持自定义提交信息模板（如Conventional Commits格式）

**增效价值**:
- 提交历史可追溯
- 避免误操作
- 团队统一提交规范

---

### 1.6 仓库分组与标签 ⭐⭐⭐
**现状问题**: 所有仓库平铺在一个列表中，无法按项目/团队/技术栈分类

**建议方案**:
- 为仓库增加"分组"和"标签"字段
- 分组：树形结构（如"公司项目/个人项目/开源贡献"）
- 标签：多标签支持（如"C#", "前端", "紧急", "归档"）
- 左侧树形导航显示分组，点击展开对应仓库
- 标签云快速过滤

**增效价值**:
- 大量仓库时保持组织性
- 快速定位特定类型项目

---

## 二、跨功能整合增强（中优先级）

### 2.1 文件搜索 → 代码工作区联动 ⭐⭐⭐⭐
**现状问题**: MFT文件搜索和代码工作区是两个独立功能，没有联动

**建议方案**:
- 在文件搜索结果中，如果文件属于某个已配置的代码仓库，显示"所属仓库"标签
- 右键菜单增加"在代码工作区中打开仓库"
- 代码工作区中增加"在此仓库中搜索文件"按钮，直接跳转到文件搜索并限定路径

**增效价值**:
- 文件搜索 → 仓库操作无缝衔接
- 快速定位文件所属项目

---

### 2.2 包管理 → 代码工作区联动 ⭐⭐⭐
**现状问题**: 包管理的产品和代码仓库可能是对应关系，但没有关联

**建议方案**:
- 在CodeRepository模型中增加`LinkedPackageId`字段
- 代码工作区中显示关联的包版本信息
- 包管理页面中增加"打开源码仓库"按钮
- 编译完成后自动触发包上传（Jenkins CI → FTP Finalize流程）

**增效价值**:
- 开发 → 编译 → 发布全流程打通
- 快速从包跳转到源码

---

### 2.3 通知中心增强 ⭐⭐⭐
**现状问题**: 通知中心只是简单的消息列表，没有分类和优先级

**建议方案**:
- 通知分类：包更新、文件传输、AI提交、系统消息
- 优先级标记：紧急/重要/普通
- 通知聚合：同类通知合并显示（如"3个包有新版本"）
- 通知操作：直接在通知中执行操作（如"立即更新"）
- 通知历史：保留最近30天，可搜索

**增效价值**:
- 重要信息不被淹没
- 减少点击次数

---

### 2.4 全局快捷键扩展 ⭐⭐⭐
**现状问题**: 只有Ctrl+Q打开快速启动，其他功能都需要鼠标点击

**建议方案**:
- 增加全局快捷键：
  - `Ctrl+Shift+C` - 打开代码工作区
  - `Ctrl+Shift+F` - 打开文件搜索
  - `Ctrl+Shift+N` - 打开通知中心
  - `Ctrl+Shift+P` - 打开包管理
- 快捷键可在设置中自定义
- 支持快捷键冲突检测

**增效价值**:
- 键盘流操作，无需鼠标
- 快速在功能间切换

---

## 三、代码工作区技术债务清理（中优先级）

### 3.1 代码重构 ⭐⭐⭐⭐
**现状问题**: 
- `RefreshProjectFiles()`, `EnumerateProjectFiles()`, `NormalizePath()` 在两个页面中重复
- 628行的CodeWorkspacePage.xaml.cs混合UI和业务逻辑
- 无MVVM分层

**建议方案**:
- 提取共享逻辑到`CodeWorkspaceService`
- 引入ViewModel层（`CodeWorkspaceViewModel`, `CodeRepositoryViewModel`）
- 使用`CommunityToolkit.Mvvm`简化MVVM实现
- 单元测试覆盖核心逻辑

**增效价值**:
- 代码可维护性提升
- 减少bug
- 便于后续扩展

---

### 3.2 异步操作改造 ⭐⭐⭐⭐
**现状问题**: 所有Git/SVN命令、文件扫描都是同步的，大仓库会卡UI

**建议方案**:
- 所有外部进程调用改为`async/await`
- 文件扫描使用后台线程
- 增加进度指示器（加载动画）
- 支持取消长时间操作

**增效价值**:
- UI永不卡顿
- 用户体验提升

---

### 3.3 解耦CommonStartup依赖 ⭐⭐⭐
**现状问题**: IDE路径解析依赖CommonStartup功能，耦合度高

**建议方案**:
- 创建独立的`IdePathResolver`服务
- 支持多种解析策略：
  1. 从CommonStartup读取（保持兼容）
  2. 从注册表读取（VS, Rider）
  3. 从PATH环境变量读取
  4. 从标准安装路径扫描
- 在设置中增加"IDE路径配置"页面，独立管理

**增效价值**:
- 降低功能间耦合
- 新用户无需先配置CommonStartup

---

## 四、新功能建议（低优先级，但高价值）

### 4.1 代码片段管理器 ⭐⭐⭐⭐⭐
**功能描述**: 
- 管理常用代码片段（C#, XAML, PowerShell, SQL等）
- 支持分类、标签、搜索
- 一键复制到剪贴板
- 支持变量占位符（如`${ClassName}`, `${Author}`）
- 支持从文件/URL导入片段
- 团队共享片段（通过FTP或Git）

**增效价值**:
- 避免重复编写样板代码
- 团队代码风格统一
- 新人快速上手

---

### 4.2 数据库连接管理 ⭐⭐⭐⭐
**功能描述**:
- 管理常用数据库连接（SQL Server, MySQL, PostgreSQL, SQLite）
- 快速打开数据库工具（SSMS, DataGrip, DBeaver）
- 连接字符串加密存储
- 支持SSH隧道连接
- 快速执行常用查询

**增效价值**:
- 统一管理所有数据库连接
- 避免到处找连接字符串

---

### 4.3 API 测试工具 ⭐⭐⭐⭐
**功能描述**:
- 类似Postman的轻量级HTTP客户端
- 保存常用API请求
- 环境变量支持（开发/测试/生产）
- 请求历史记录
- 响应格式化（JSON, XML）
- 批量测试（健康检查）

**增效价值**:
- 无需切换到浏览器或Postman
- 快速验证API

---

### 4.4 正则表达式测试器 ⭐⭐⭐
**功能描述**:
- 输入正则表达式和测试文本
- 实时高亮匹配结果
- 显示捕获组
- 常用正则模式库（邮箱、手机号、URL等）
- 支持C#, JavaScript, Python等不同语言的正则语法

**增效价值**:
- 快速验证正则表达式
- 避免在代码中反复试错

---

### 4.5 文本处理工具集 ⭐⭐⭐
**功能描述**:
- JSON格式化/压缩/验证
- XML格式化/验证
- Base64编解码
- URL编解码
- Hash计算（MD5, SHA1, SHA256）
- 文本差异对比
- 大小写转换、去重、排序

**增效价值**:
- 常用文本处理一站式解决
- 无需在线工具（安全）

---

### 4.6 进程监控与管理 ⭐⭐⭐
**功能描述**:
- 监控特定进程（如Revit, Visual Studio）
- 进程启动/停止通知
- 资源占用监控（CPU, 内存）
- 进程崩溃自动重启
- 进程日志收集

**增效价值**:
- 及时发现进程异常
- 自动化运维

---

### 4.7 团队协作看板 ⭐⭐⭐⭐
**功能描述**:
- 轻量级任务看板（类似Trello）
- 本地存储或团队共享（通过FTP/Git）
- 任务状态：待办/进行中/已完成
- 任务分配、优先级、标签
- 与PingCode集成（双向同步）

**增效价值**:
- 小团队无需购买项目管理工具
- 与现有PingCode集成

## 七、技术选型建议

### 推荐引入的库
- **CommunityToolkit.Mvvm** - 简化MVVM实现
- **LibGit2Sharp** - Git操作的.NET库（比调用git.exe更可靠）
- **SharpSvn** - SVN操作的.NET库
- **Newtonsoft.Json** (已有) - JSON处理
- **Serilog** - 结构化日志（替代当前的LoggingService）
- **Polly** - 重试策略（网络请求、文件操作）
- **FluentValidation** - 数据验证

### 架构改进
- 引入依赖注入容器（Microsoft.Extensions.DependencyInjection）替代ServiceLocator
- 使用MediatR实现功能间松耦合通信
- 引入单元测试框架（xUnit + Moq）

## 九、风险与挑战

### 技术风险
- LibGit2Sharp/SharpSvn可能与现有脚本冲突 → 保留脚本作为fallback
- 异步改造可能引入并发bug → 充分测试
- MVVM重构工作量大 → 分模块渐进式重构

### 兼容性风险
- 新功能可能影响现有用户习惯 → 保留旧功能，新功能可选
- 配置文件格式变更 → 提供迁移工具

### 资源风险
- 开发时间投入 → 按优先级分阶段实施
- 测试覆盖 → 重点功能优先测试

---

## 十、总结

代码工作区当前是一个"启动器"，通过以上增强可以升级为"开发中枢"：
- **状态可视化** - 所有仓库状态一目了然
- **操作便捷化** - 常用操作一键完成
- **流程自动化** - 开发→编译→发布全流程打通
- **工具集成化** - 代码片段、API测试、数据库等工具一站式

最终目标：**让开发者80%的日常操作都可以在DevKit中完成，无需频繁切换工具。**
