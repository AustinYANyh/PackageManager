# 文件搜索与常用启动项窗口交互与快捷键设计

## 1. 文档信息

- 文档名称：文件搜索与常用启动项窗口交互与快捷键设计
- 文档类型：交互设计文档
- 对应需求文档：`docs/requirements/file-search-and-common-startup-prd.md`
- 文档目标：明确两个窗口的核心交互结构、键盘路径、鼠标路径和高频动作表现

## 2. 设计目标

本设计不是单纯增加快捷键，而是要让两个窗口都具备以下特征：

- 打开后无需思考，立刻可以操作
- 鼠标可完成全部核心动作
- 键盘可完成全部高频动作
- 搜索、选择、处理形成连续闭环
- 关闭、隐藏、返回、编辑等行为保持统一

## 3. 整体交互原则

### 3.1 打开窗口后的默认行为

- 窗口显示后，默认聚焦搜索框
- 搜索框内容默认全选，用户可直接覆盖输入
- 若窗口已有历史内容，则保留上次搜索结果，但焦点仍回到搜索框
- 若通过快捷键再次唤起窗口，优先表现为“恢复并聚焦”，而不是重新创建实例

### 3.2 键盘优先原则

- 用户输入关键词后，不需要切换焦点即可开始筛选
- 搜索结果出现后，支持方向键进入结果区
- 进入结果区后，主要动作通过 `Enter / Ctrl+Enter / Ctrl+C / Esc` 完成
- 所有高频动作应尽量有显式快捷键，不依赖右键菜单

### 3.3 关闭与隐藏原则

- `Esc` 与点击右上角 `X` 保持一致
- 非退出场景下，关闭表现为隐藏窗口
- 真正退出仅发生在宿主进程退出或明确的进程关闭流程中

### 3.4 结果操作原则

- 选中一个结果后，所有可执行动作必须可见
- 双击和 `Enter` 对应“主动作”
- 右键菜单承载低频动作
- 工具栏按钮承载高频、可见动作

## 4. 文件搜索窗口交互设计

## 4.1 窗口结构

建议采用三段式结构：

1. 顶部搜索区
2. 中部结果区
3. 底部状态与动作区

### 4.1.1 顶部搜索区

包含以下元素：

- 搜索输入框
- 快速过滤切换
  - 全部
  - 可启动文件
  - 文件夹
  - 脚本
  - 日志
  - 配置
- 范围选择
  - 全盘
  - 常用目录
  - 当前工作区
  - 自定义范围
- 重建索引按钮
- 高级语法提示入口

### 4.1.2 中部结果区

采用列表或表格视图，建议字段如下：

- 名称
- 路径
- 类型
- 修改时间
- 大小
- 标记状态

结果区顶部建议保留轻量结果工具条：

- 打开
- 打开所在目录
- 复制路径
- 加入常用启动项
- 更多操作

### 4.1.3 底部状态区

展示以下信息：

- 当前匹配数量
- 索引状态
- 当前过滤条件
- 可用快捷键提示

建议状态栏右侧显示简版提示：

- `Enter 打开`
- `Ctrl+Enter 打开目录`
- `Ctrl+C 复制路径`
- `Esc 隐藏`

## 4.2 文件搜索主交互流程

### 4.2.1 标准流程

1. 用户按 `Ctrl+E` 打开文件搜索
2. 焦点进入搜索框，原关键词全选
3. 用户输入关键词
4. 结果实时刷新
5. 用户通过方向键或鼠标选中结果
6. 用户执行后续动作
   - 打开
   - 打开目录
   - 复制路径
   - 加入常用启动项
7. 窗口根据动作配置决定是否保持显示

### 4.2.2 “搜索后直接打开”路径

- 输入关键词
- `Down` 进入首个结果
- `Enter` 打开

这是最核心的 3 步闭环，应优先保证流畅。

### 4.2.3 “搜索后加入常用启动项”路径

- 输入关键词
- `Down` 进入结果
- `Ctrl+D` 或专用快捷键加入常用启动项
- 右下角轻提示反馈“已加入”

### 4.2.4 “搜索后复制路径”路径

- 输入关键词
- 选择结果
- `Ctrl+C`
- 底部状态栏显示“路径已复制”

## 4.3 文件搜索快捷键设计

### 4.3.1 全局入口

- `Ctrl+E`
  - 打开或唤醒文件搜索窗口
  - 仅在允许用户下生效

### 4.3.2 窗口级快捷键

- `Esc`
  - 隐藏窗口
- `Ctrl+L`
  - 聚焦搜索框并全选当前关键词
- `F5`
  - 刷新当前结果
- `Ctrl+R`
  - 重建索引
- `Tab`
  - 在搜索区、过滤区、结果区间切换焦点
- `Shift+Tab`
  - 逆向切换焦点

### 4.3.3 搜索框快捷键

- `Enter`
  - 若结果为空：保留当前焦点
  - 若已有选中结果：执行主动作
  - 若尚未选中结果但有结果：默认选中第一项并打开
- `Down`
  - 从搜索框进入结果区第一项
- `Up`
  - 从搜索框进入结果区最后一项
- `Ctrl+Backspace`
  - 删除一个词段
- `Ctrl+A`
  - 全选搜索词

### 4.3.4 结果区快捷键

- `Up / Down`
  - 上下移动结果选择
- `PageUp / PageDown`
  - 翻页移动
- `Home / End`
  - 跳到首项 / 末项
- `Enter`
  - 打开选中项
- `Ctrl+Enter`
  - 打开所在目录
- `Shift+Enter`
  - 管理员方式运行
- `Ctrl+C`
  - 复制完整路径
- `Ctrl+Shift+C`
  - 复制文件名
- `Alt+Enter`
  - 查看属性
- `Ctrl+D`
  - 加入常用启动项
- `Ctrl+T`
  - 在终端中打开当前目录
- `F2`
  - 重命名
- `Delete`
  - 删除或移入回收站

## 4.4 文件搜索鼠标交互设计

- 单击：选中结果
- 双击：执行主动作“打开”
- 右键：弹出上下文菜单
- 中键：可选设计为“打开所在目录”

### 4.4.1 结果右键菜单建议

- 打开
- 打开所在目录
- 复制完整路径
- 复制文件名
- 复制父目录路径
- 以管理员身份运行
- 用指定程序打开
- 打开终端到当前目录
- 加入常用启动项
- 查看属性

## 4.5 文件搜索交互细节建议

### 4.5.1 无结果状态

- 文案不只显示“无结果”
- 应显示当前范围和过滤条件
- 给出推荐动作
  - 清空过滤器
  - 扩大范围
  - 改用文件夹模式

### 4.5.2 索引未就绪状态

- 可先显示已有可用结果或加载状态
- 避免整个窗口完全不可操作
- 状态栏明确提示“索引初始化中”

### 4.5.3 操作反馈

- 复制成功
- 加入成功
- 启动成功
- 启动失败
- 路径失效

反馈建议使用轻提示，不频繁弹阻断式对话框。

## 5. 常用启动项窗口交互设计

## 5.1 窗口结构

建议继续保留左右双栏结构，但强化角色分工。

### 左侧：搜索结果区

作用：

- 临时查找候选启动项
- 作为“加入启动项”的来源区域

### 右侧：常用启动项区

作用：

- 展示当前已保存启动项
- 承担快速启动、编辑、删除、排序等核心操作

### 顶部：统一搜索区

包含：

- 搜索框
- 重建索引
- 当前快捷键提示

### 底部：动作区

包含：

- 添加所选
- 手动添加
- 启动
- 编辑
- 删除
- 关闭

## 5.2 常用启动项主交互流程

### 5.2.1 “搜索并加入”路径

1. 打开常用启动项窗口
2. 输入关键词
3. 在左侧搜索结果中选中项
4. 双击或快捷键加入
5. 右侧即时出现新增项
6. 用户可继续编辑或直接启动

### 5.2.2 “直接启动常用工具”路径

1. 按 `Ctrl+Q` 打开窗口
2. 搜索框全选
3. 用户输入名称过滤右侧或直接用方向键选择右侧项
4. `Enter` 启动
5. 根据配置选择是否自动隐藏窗口

### 5.2.3 “维护启动项”路径

1. 打开窗口
2. 定位右侧项
3. `F2` 编辑
4. `Delete` 删除
5. `Ctrl+Up / Ctrl+Down` 调整顺序

## 5.3 常用启动项快捷键设计

### 5.3.1 全局入口

- `Ctrl+Q`
  - 打开或唤醒常用启动项窗口

### 5.3.2 窗口级快捷键

- `Esc`
  - 隐藏窗口
- `Ctrl+L`
  - 聚焦搜索框并全选
- `Tab`
  - 在搜索框、左侧候选区、右侧常用项区间切换
- `Shift+Tab`
  - 逆向切换

### 5.3.3 左侧候选区快捷键

- `Up / Down`
  - 移动选中项
- `Enter`
  - 加入常用启动项
- `Ctrl+Enter`
  - 打开候选项所在目录
- `Ctrl+C`
  - 复制候选项路径
- `Ctrl+D`
  - 加入常用启动项

### 5.3.4 右侧常用项区快捷键

- `Up / Down`
  - 移动选中项
- `Enter`
  - 启动选中项
- `Ctrl+Enter`
  - 打开所在目录
- `Shift+Enter`
  - 管理员启动
- `F2`
  - 编辑
- `Delete`
  - 删除
- `Ctrl+C`
  - 复制路径
- `Ctrl+Up`
  - 上移排序
- `Ctrl+Down`
  - 下移排序
- `Ctrl+D`
  - 复制一条启动项配置

## 5.4 常用启动项鼠标交互设计

### 左侧候选区

- 单击：选中
- 双击：加入常用启动项
- 右键：显示候选项操作菜单

### 右侧常用项区

- 单击：选中
- 双击：启动
- 右键：显示启动项操作菜单
- 拖拽：后续支持排序调整

### 右侧菜单建议

- 启动
- 管理员启动
- 打开所在目录
- 编辑
- 复制路径
- 上移
- 下移
- 删除

## 5.5 常用启动项交互细节建议

### 5.5.1 启动反馈

- 启动时右下角提示“已启动”
- 启动失败时显示失败原因
- 若路径失效，给出“重新定位”入口

### 5.5.2 编辑反馈

- 编辑保存后就地刷新，不弹多余提示
- 若修改路径，建议校验是否存在

### 5.5.3 加入反馈

- 从左侧加入右侧后，右侧新增项短暂高亮
- 重复加入时提示“已存在”

## 6. 两个窗口的统一交互规范

为了降低学习成本，建议统一以下行为：

### 6.1 统一快捷键

- `Esc`：隐藏
- `Ctrl+L`：聚焦搜索框
- `Enter`：执行主动作
- `Ctrl+Enter`：打开所在目录
- `Ctrl+C`：复制路径
- `F2`：编辑或重命名
- `Delete`：删除

### 6.2 统一状态反馈

- 成功动作用轻提示
- 异常动作用错误提示
- 不对高频成功动作弹 MessageBox

### 6.3 统一焦点策略

- 窗口打开默认焦点在搜索框
- 从搜索框可直接 `Down` 进入列表
- 焦点丢失后再次唤醒，恢复到上次操作上下文

## 7. 推荐的第一阶段落地交互

如果只做一版最有感知的第一阶段交互，建议优先实现以下内容：

### 文件搜索

- `Ctrl+E` 唤起
- 搜索框默认全选
- `Down` 进入结果区
- `Enter` 打开
- `Ctrl+Enter` 打开所在目录
- `Ctrl+C` 复制路径
- `Ctrl+D` 加入常用启动项
- `Esc` 隐藏
- 右键菜单提供基础动作

### 常用启动项

- `Ctrl+Q` 唤起
- `Enter` 启动
- `Ctrl+Enter` 打开所在目录
- `F2` 编辑
- `Delete` 删除
- `Esc` 隐藏
- 左侧双击加入
- 右侧双击启动

## 8. 交互效果预期

做到以上设计后，用户对两个窗口的感受应该是：

- 打开就能用，不需要重新找输入框
- 不依赖鼠标也能完成大部分高频动作
- 搜索结果不是终点，而是处理入口
- 常用启动项不再只是列表，而是一个真正可依赖的启动面板
- 两个窗口学一个就能迁移到另一个
